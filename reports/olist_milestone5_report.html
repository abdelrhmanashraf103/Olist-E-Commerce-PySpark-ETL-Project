Key Design Decisions
  • D-01
    Parquet over CSV for Output — Columnar storage speeds up queries by 5–10x and reduces file size by ~70%.
  • D-02
    Partition by Year/Month — Partition pruning when querying a specific time range avoids a full data scan.
  • D-03
    Modular Functions — Each table has an independent clean_X() function for easy testing, maintenance, and isolated modification.
  • D-04
    Aggregate Before Join — Aggregating Items and Payments at the Order level first reduces data volume and avoids a Cartesian Product.
  • D-05
    JSON Checkpoint for Incremental — Storing the last run time in a simple JSON file minimizes dependencies and simplifies debugging.
  • D-06
    Window Function for Dedup — row_number() with partitionBy selects the latest review per order with higher precision than dropDuplicates().
  • D-07
    Brazil Bounding Box — Filtering coordinates outside Brazil's geographic bounds removes corrupt data without needing an external API.
  • D-08
    shuffle.partitions=8 — Optimized for the local environment. In production, raise to 200+ based on data volume and core count.
